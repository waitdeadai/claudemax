# claudemax vs the 2026 SOTA landscape

> Live-research synthesis as of 2026-05-20. Sources cited inline. This page is the result of `/deepresearch` against current Anthropic docs and the 2026 multi-agent coding ecosystem.

## TL;DR

For a Claude Max subscriber wanting **CLI-first, spec-driven, max-parallel** autonomous coding with **independent verification** and **honesty-gated outputs**, claudemax v0.2 implements the 2026 SOTA pattern with these distinguishing properties:

- Wraps **Claude Code's native `/goal` validator-loop** (v2.1.139, May 12 2026) — does not reimplement.
- **Multispec engine as default behavior** of every umbrella — matches the [Task-Decoupled Planning (TDP) pattern](#sources) (supervisor → DAG decomposition → executor → self-revision).
- **Two parallelism modes auto-selected** — SDK subagents (Mode A) for small/short, [Claude Code Agent Teams](https://code.claude.com/docs/en/agent-teams) (Mode B, experimental v2.1.32) for big swarms.
- **Plan-aware cost-guard** tuned for Claude Max — first harness I could find that auto-detects Max5x/Max20x and gates the router against the Agent SDK credit pool (separate from interactive usage since the [June 15 2026 billing split](https://help.apiyi.com/en/anthropic-claude-subscription-agent-sdk-billing-split-june-2026-en.html)).
- **Bundled dark-patterns** (35 honesty hooks from [waitdeadai/llm-dark-patterns](https://github.com/waitdeadai/llm-dark-patterns)) — no separate plugin install.
- **Subscription-compliant** — routes all `query()` calls through `@anthropic-ai/claude-agent-sdk` which bills against the Agent SDK credit pool. NOT affected by the [April 4 2026 OpenClaw block](https://thenextweb.com/news/anthropic-openclaw-claude-subscription-ban-cost) (which Anthropic [reversed](https://www.datagrom.com/ai-news/anthropic-reverses-ban-on-third-party-ai-agent-use-8ec3aaa6) by introducing the Agent SDK credit subcategory we use).
- **Remote-from-phone** via Tailscale + tmux + ntfy with phone-side QR onboarding — no comparable polish in competing harnesses.

## Where claudemax sits in the 2026 ecosystem

Three layers of multi-agent orchestration emerged in 2026 per [Daniel Vaughan's analysis](https://codex.danielvaughan.com/2026/04/09/claude-multi-agent-ecosystem/):

1. **Anthropic-official**: [Agent Teams](https://code.claude.com/docs/en/agent-teams) (experimental, shipped with Opus 4.6, refined in Opus 4.7 release April 16 2026), [Managed Agents](https://platform.claude.com/docs/en/managed-agents/overview) (hosted REST API).
2. **Community OSS orchestrators**: [Ruflo (formerly claude-flow)](https://pasqualepillitteri.it/en/news/774/claude-flow-ruflo-multi-agent-orchestration-guide) — 31,100+ stars, 84.8% SWE-bench, "75% API cost savings via caching"; [wshobson/agents](https://github.com/wshobson/agents) — 80 plugins, 185 agents, 153 skills; Claude Squad; OpenClaw+Antfarm.
3. **Power-user harnesses**: claudemax, [Forge](https://www.augmentcode.com/tools/open-source-agent-orchestrators) (three-command spec pipeline for Claude Code), [Composio AO](https://www.augmentcode.com/guides/how-to-run-a-multi-agent-coding-workspace) (full automation with isolated worktrees per agent).

claudemax fits layer 3. It is **not** a Ruflo competitor (Ruflo is broader, multi-model, multi-IDE; claudemax is intentionally Anthropic-CLI-only). It is **not** a Managed Agents replacement (those run on Anthropic's hosted sandbox; claudemax runs locally). It is the **lean power-user CLI** for someone who lives in Claude Code and wants a spec-driven autonomy substrate without giving up control to a hosted system.

## Where claudemax is genuinely SOTA

**1. Plan-aware cost-guard for Claude Max subscribers.**
Per my [Anthropic billing-split research](https://help.apiyi.com/en/anthropic-claude-subscription-agent-sdk-billing-split-june-2026-en.html), the Agent SDK credit pool exists ($100 Max5x, $200 Max20x) but no harness I could find auto-detects the plan and gates the router against the credit consumption with 70/90/95% thresholds. claudemax's `packages/core/src/router.ts` does this; `cmax doctor` shows it live.

**2. Dark-patterns bundled-by-default.**
The waitdeadai/llm-dark-patterns repo (your own work) catches 35 distinct LLM dark patterns (vibes, emoji spam, aggregator hallucination, fake citations, no-credential-leak-in-handoff). I could not find another harness that bundles honesty-gating hooks by default. Most either skip honesty entirely or rely on the model behaving well.

**3. Two parallelism modes auto-selected per spec shape.**
Mode A vs Mode B is decided by the multispec engine based on sub-spec count, est. duration, cross-spec coordination need, and write-set overlap. Other harnesses force one mode globally and let the user pick.

**4. Remote-from-phone polish.**
`cmax bg phone` prints QR codes (when `qrencode` is installed) for ntfy deep link, Tailscale download, Termius/Termux install, with the Tailscale hostname auto-interpolated into the SSH attach command. Per [Sealos 2026](https://sealos.io/blog/claude-code-on-phone/), the Tailscale + tmux + Termius stack is the community gold standard — but no harness I could find ships first-class phone-onboarding tooling.

**5. `/taste` auto-bootstrap (no 10-question wizard).**
Replaces the v1 minmaxing flow. Reads repo + /deepresearch SOTA at current time → writes taste.md + taste.vision with zero questions in the common case. Other harnesses either skip taste or use static templates.

## What the May 2026 Anthropic releases changed

Verified against [Anthropic's official Opus 4.7 release notes](https://platform.claude.com/docs/en/about-claude/models/whats-new-claude-4-7) and [models overview](https://platform.claude.com/docs/en/about-claude/models/overview):

- **Opus 4.7 pricing is $5 input / $25 output per MTok** (not the $15/$75 I had in v0.1; fixed in `packages/core/src/models.ts`).
- **Opus 4.7 has 1M context window** at standard pricing, no long-context premium.
- **Opus 4.7 has 128k max output** (up from 32k assumption).
- **`xhigh` effort level is new** and is Anthropic's recommended default for coding/agentic use. claudemax exposes this via the runtime's `effort` option (currently defaults to `'max'`; users can override via `--effort xhigh`).
- **Extended thinking budgets removed in Opus 4.7** — `thinking: {type: "enabled", budget_tokens: N}` returns 400. Use `thinking: {type: "adaptive"}` (off by default; must opt in).
- **Sampling parameters (`temperature`, `top_p`, `top_k`) removed in Opus 4.7** — return 400 if set. claudemax doesn't set them.
- **`task_budget` (beta)** — advisory token budget across full agentic loop. Beta header `task-budgets-2026-03-13`. Maps directly to claudemax's `--budget-credits` flag conceptually. Wireable via the SDK's `betas` array — see `docs/MULTISPEC.md` for the integration point.

## Claude Agent SDK options claudemax adopts vs leaves on the table

Verified against [Claude Code Agent SDK TS reference](https://code.claude.com/docs/en/agent-sdk/typescript):

**Adopted:**
- `model`, `fallbackModel` — primary + fallback
- `effort: 'max'` (will move to 'xhigh' in a follow-up)
- `systemPrompt: { type: 'preset', preset: 'claude_code', append }` — inherits Claude Code's tool surface + caching
- `allowedTools`, `permissionMode` — fine-grained tool gating
- `settingSources: ['user', 'project']` — loads CLAUDE.md, project skills, slash commands
- `skills: 'all'` — exposes installed skill bundle
- `enableFileCheckpointing: true` — rewind capability
- `agentProgressSummaries: true`, `forwardSubagentText: true` — subagent observability
- `abortController` — SIGINT-driven cancellation
- `maxBudgetUsd` — hard-stop budget cap
- `cwd`, `env` — process control
- `resume` — session resumption (used by /overnight)
- `outputFormat: { type: 'json_schema', schema }` — structured output for spec/verify
- `mcpServers` — MCP integration point (configurable per-project)
- `hooks` — callback wiring

**Not yet adopted (open follow-ups, tracked in CHANGELOG):**
- `includeHookEvents: true` — emit hook lifecycle in stream (new May 2026)
- `strictMcpConfig: true` — stricter MCP validation (new May 2026)
- `sessionStoreFlush: 'eager'` — near-real-time transcript mirror for live-tailing (new May 2026)
- `betas: ['task-budgets-2026-03-13']` + `taskBudget: { total }` — advisory token budget across the agentic loop (Opus 4.7 beta)
- `thinking: { type: 'adaptive' }` — explicit adaptive thinking on Opus 4.7 (off by default in 4.7)
- `context: 'fork'` in skill frontmatter — hot-reload skill in forked subagent context (new May 2026)

## Compliance posture

**April 4 2026 block of OpenClaw / claude-flow on Pro/Max subscriptions**: this was [reversed](https://www.datagrom.com/ai-news/anthropic-reverses-ban-on-third-party-ai-agent-use-8ec3aaa6) when Anthropic introduced the Agent SDK credit subcategory specifically for programmatic / third-party tool usage. claudemax has always routed through `@anthropic-ai/claude-agent-sdk` `query()` — the official, compliant path that bills against the Agent SDK credit pool (separate from interactive usage limits).

Users opt into the API key billing path (`CMAX_PLAN=api` or `ANTHROPIC_API_KEY` set) explicitly. Default is subscription via the Agent SDK credit pool.

## Open research questions

1. **Cache hit rate measurement**: The SDK reports `cache_read_input_tokens` and `cache_creation_input_tokens` in `result.usage`. There's a [known bug](https://github.com/anthropics/claude-agent-sdk-typescript/issues/188) where the SDK marks content as fresh ephemeral writes per turn even when cached, paying 1.25× write rate instead of 0.1× read rate. claudemax should surface cache hit rate to detect when the bug bites your workload — wired in `packages/core/src/cost.ts:cacheStatsFromUsage`; CLI surface via `cmax memory credit` is a follow-up.

2. **task_budget vs maxBudgetUsd**: Anthropic shipped `task_budget` in Opus 4.7 as a beta-gated advisory budget the model is AWARE of (it sees a running countdown). claudemax uses `maxBudgetUsd` (hard cap, model unaware). Both are useful; the former produces more graceful task-shortening, the latter prevents overrun. Wire both per-umbrella for v0.3.

3. **`xhigh` vs `max` effort on Opus 4.7**: Anthropic recommends `xhigh` as the default for coding. claudemax currently defaults to `max`. Difference: `max` uses more tokens for the same task; `xhigh` is the calibrated agentic-coding sweet spot per the docs. v0.3 should default to `xhigh` and surface `--effort max` as the deliberate-override path.
